# PRD

## プロジェクト名

**SOC-Lab：Dockerベース統合セキュリティ学習・SOC訓練環境**

---

## 1. 背景・目的（Why）

### 背景

* セキュリティ学習は「ツール単体の使い方」で止まりがち
* 実務では

  * ネットワーク
  * アプリ
  * OS
  * ログ相関
    を横断して判断する **SOC的視点** が必須
* しかし、それを **安全かつ再現可能に学べる環境** が少ない

### 課題

* IDS / WAF / HIDS / SIEM が **分断** されて学ばれる
* 「攻撃が成功しているのか」「防御できているのか」が体感しづらい
* ログを“見て終わり”で **判断・改善サイクル** まで到達しない

### 目的

* Dockerで完結する **実戦型SOC訓練環境** を提供
* **攻撃 → 検知 → 対応 → 報告 → 改善** の一連を体験
* **OSIレイヤーごとに何が起きているかを可視化** し、立体的に理解する

---

## 2. プロダクトゴール（What）

### ゴール

* ユーザーが以下を **説明・再現・改善** できる状態になる

  * 攻撃が「どのレイヤーで」「どのログに」現れるか
  * 検知できていない場合、**どこをどう直すか**
  * 検知後、**SOCとして何を判断・報告・改善するか**

### 非ゴール

* 商用WAF/EDRの完全再現
* 高度なゼロデイ研究
* クラウド特化（AWS GuardDuty等）

## 4. 全体アーキテクチャ

```
[ Kali Linux ]        (攻撃)
      |
      v
[ NestJS App ]        (L7)
      |
      +--> App Log --> Fail2ban (L7防御)
      |
      +--> Network --> Suricata (L3-7検知)
      |
[ Linux Host ]
      |
      +--> Auditd (L7/OS監査)
      |
[ Filebeat ]
      |
[ Elasticsearch ] <---> [ Kibana ]
        (SIEM / SOC)
```

## 8. 各シナリオ詳細（例）

### S2：APIブルートフォース

#### 攻撃

* Kali → hydra → `/login`

#### 発生事象

| レイヤー | 観測点                           |
| ---- | ----------------------------- |
| L7   | `/var/log/app/auth.log` に失敗ログ |
| 防御   | Fail2banがIPをBAN               |
| SIEM | Kibanaで失敗→BANの相関              |

#### 成功判定

* ❌ 未防御：無限に試行可能
* ✅ 防御：iptables DROP / タイムアウト

#### 改善点

* failregex調整
* maxretry最適化
* レート制限強化

## 9. SOC訓練フロー（How to operate）

### フェーズ

1. **検知（Detect）**

   * Suricata alert
   * Fail2ban ban
   * Auditd event
2. **分析（Analyze）**

   * いつ / 誰が / どこから / 何を
3. **対応（Respond）**

   * BAN / 遮断 / 設定変更
4. **報告（Report）**

   * 影響範囲
   * 原因
5. **改善（Improve）**

   * ルール修正
   * ダッシュボード強化
