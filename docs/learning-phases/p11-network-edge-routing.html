Advanced Network・Edge Routing

P12に進む前に、network障害をL3/L4/L7/edgeに分解して説明できる

SREWhitehat

抽象的に何を学ぶか

Advanced networkでは、packet単体ではなく、clientからedge、load balancer、service、resolverまでの経路全体を障害ドメインとして扱います。

具体例

  • TCP: SYN backlogや再送が増えるとHTTP 5xxではなく接続遅延として現れる
  • TLS/mTLS: 証明書期限、SNI、cipher policy、client certificate失敗をHTTP障害と分ける
  • BGP/Anycast/CDN: 実BGPは触らず、edge routingの切替判断と観測点を図にする

到達目標

  • TCP再送、SYN backlog、timeoutをSLO劣化として説明できる
  • TLS/mTLSのhandshake failureを証明書、SNI、protocol、policyに分解できる
  • DNS、CDN、Anycast、L4/L7 LBの責務と観測ログを説明できる

カバー領域

  • TCP retransmission
  • SYN backlog
  • TLS handshake
  • mTLS
  • DNS resolver
  • BGP
  • Anycast
  • QUIC
  • HTTP/3
  • L4/L7 load balancer
  • CDN edge routing

Hands-on Flow

  • edge proxy profileを起動し、app直通とproxy経由のhealthを比較する
  • openssl s_clientでTLS endpointがない場合の失敗と、TLS終端がある場合に必要な証跡を整理する
  • DNS resolver、edge、originの障害切り分け表を作る

合格証跡

  • direct/proxy/DNSの切り分け結果がある
  • TLS/mTLSで必要な証跡と監視項目を列挙している
  • edge routing障害時のrollbackまたはtraffic shift判断がある
