抽象的に何を学ぶか

倫理・安全範囲・作業環境では、個別のコマンド暗記ではなく、第三者環境を攻撃しない制約を説明できるための判断軸を学びます。

具体例

  • 具体例: README、SECURITY、CONTRIBUTINGを読み、許可された閉域ラボだけを対象にする
  • 具体例: feature branchを作り、ローカル品質ゲートの軽量チェックを実行する
  • 具体例: incident reportとvulnerability remediation PRのテンプレートを埋める

到達目標

  • 第三者環境を攻撃しない制約を説明できる
  • clone、branch、test、PR、CI確認までを一人で実行できる
  • レポートやログから秘密情報を除外できる

カバー領域

  • 許可範囲
  • GitHub Flow
  • Docker基礎
  • 証跡管理

Hands-on Flow

合格証跡

  • 対象、日時、実行コマンド、影響範囲を記録している
  • 秘密情報や外部ターゲットを含まない学習ログになっている
  • GitHub FlowでPR化できる
