抽象的に何を学ぶか

Linux・Docker・ネットワーク基礎では、個別のコマンド暗記ではなく、container、network namespace、port publish、volumeの違いを説明できるための判断軸を学びます。

具体例

  • 具体例: base stackを起動し、app、db、suricata、kaliのnetworkを確認する
  • 具体例: ping、traceroute、ss、tcpdumpで疎通を確認する
  • 具体例: nmapとSuricataログを対応付ける

到達目標

  • container、network namespace、port publish、volumeの違いを説明できる
  • 疎通障害をDNS、TCP、HTTPの順に切り分けられる
  • S1、S8、S9、S10でL2-L4の観測境界を理解する

カバー領域

  • Linux process
  • namespace
  • container network
  • tcpdump
  • ss

Hands-on Flow

  • base stackを起動し、app、db、suricata、kaliのnetworkを確認する
  • ping、traceroute、ss、tcpdumpで疎通を確認する
  • nmapとSuricataログを対応付ける

scripts/learning_phase.sh config p1

実行コマンド

docker compose up -d --build
docker compose ps
docker exec -it soc-lab-kali /scripts/s1_portscan.sh
docker exec soc-lab-suricata grep SCAN /var/log/suricata/fast.log | tail -20

合格証跡

  • container IP、published port、service name resolutionを説明している
  • tcpdumpまたはSuricata fast.logで通信を確認している
  • ネットワーク障害とアプリ障害を分けて記録している
