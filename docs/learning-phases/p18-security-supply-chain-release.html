P18 / Lead 11 / phase18-security-supply-chain

Security Engineering・Supply Chain・Release

P19に進む前に、コードからartifact/release/開示まで一気通貫で説明できる

WhitehatBackendSRE

抽象的に何を学ぶか

Security supply chainでは、コード脆弱性だけでなく、依存、build、artifact、secret、release経路まで攻撃面として扱います。

具体例

  • SSRF/BOLA/upload: business logicとauthorizationの抜けをred testで固定する
  • SBOM/SAST/DAST/SCA: 依存とコードと実行中APIを別々の観点で検査する
  • artifact signing/release: どのcommitからどのimageが出たかを検証可能にする

到達目標

  • OWASP API Top 10の上位リスクをhands-on testへ変換できる
  • SBOM、SAST、DAST、SCA、secret scanning、artifact signingの役割を説明できる
  • GitOps、feature flag、blue-green/canary、rollback safetyをrelease gateにできる

Hands-on Flow

  • npm audit、OpenAPI contract、backend hands-onをまとめてsecurity gateとして実行する
  • SSRF/BOLA/upload/RCEの安全なred test設計をHTMLに沿って作る
  • release checklistで署名、SBOM、rollback、migration互換性を確認する
