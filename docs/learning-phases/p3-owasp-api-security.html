P3 / Operator 1 / base

OWASP API Security・認証認可・安全な失敗

P4に進む前に、攻撃の成功/失敗理由をコードとログで説明できる

WhitehatBackend

到達目標

  • SQLi、path traversal、認証失敗、認可失敗を安全に再現できる
  • 攻撃結果、アプリログ、検知ログ、修正テストを一つの証跡にできる
  • IDOR/BOLA、SSRF、unsafe uploadなど未実装領域を課題化できる

カバー領域

  • BOLA
  • auth hardening
  • SQLi
  • path traversal
  • rate limit

Hands-on Flow

  • S2、S3を実行して認証攻撃とSQLi試行を検知する
  • 失敗時にcredential materialやstack traceが漏れないことを確認する
  • 追加課題としてBOLA/SSRF/Uploadのred test設計を書く
