Infrastructure as Code・Policy as Code

P15に進む前に、infra変更をpolicyとevidenceで制御できる

SREWhitehat

抽象的に何を学ぶか

IaCでは、環境を手で直すのではなく、変更意図、plan、policy、state、driftをレビュー可能な証跡にします。

具体例

  • Terraform module: prod/stageで同じmoduleを使い、変える値だけを入力にする
  • drift detection: console手変更を検出し、緊急対応後もcodeへ戻す
  • OPA/Conftest: public bucketや0.0.0.0/0 ingressをCIで拒否する

到達目標

  • Terraform module設計、state管理、環境分離、drift検知を説明できる
  • OPA/Conftest相当のpolicy-as-codeで危険なplanを拒否できる
  • CIでplan review、security review、rollback reviewを分けられる

カバー領域

  • Terraform module
  • state management
  • drift detection
  • policy-as-code
  • OPA
  • Conftest
  • CI plan review
  • environment separation

Hands-on Flow

  • サンプルTerraform差分を読み、危険なpublic exposureやstate分離不足を指摘する
  • policy check scriptでIaC guardrailを検証する
  • plan review commentのテンプレートを作る

  • plan reviewで拒否すべき差分と許可できる差分を分けている
  • state、drift、environment separationの運用リスクを説明している
  • policy-as-codeの例外申請と期限を定義している
