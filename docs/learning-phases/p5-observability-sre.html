Observability・SLO・Incident Command

P6に進む前に、障害対応を再現手順と運用判断で説明できる

SREBackend

到達目標

  • health、readiness、blackbox probe、latencyをSLO証跡へ変換できる
  • incident commander、scribe、operatorの役割でS14を実施できる
  • メトリクス、ログ、トレースの使い分けを説明できる

カバー領域

  • Prometheus
  • Grafana
  • OpenTelemetry
  • blackbox probe
  • burn-rate

Hands-on Flow

  • observability profileを起動し、PrometheusとGrafanaを確認する
  • blackbox probeでapp healthを監視する
  • load_hands_on_testsとincident_drillの結果をpostmortemへ入れる

Docker実行

scripts/learning_phase.sh config p5
scripts/learning_phase.sh start p5
scripts/learning_phase.sh status p5

実行コマンド

scripts/learning_phase.sh start p5
curl -s http://localhost:9090/-/ready
scripts/load_hands_on_tests.sh
RUN_CHAOS=1 scripts/incident_drill.sh

合格証跡

  • SLO、SLI、error budget、burn-rate条件を文章化している
  • PrometheusまたはGrafanaで可用性/レイテンシを確認している
  • MTTD/MTTRと暫定対応/恒久対応をpostmortemに残している
