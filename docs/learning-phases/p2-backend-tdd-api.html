BackendSRE

到達目標

  • API contract、unit test、DB integration testの役割を分けられる
  • 入力検証、エラー形状、DB境界をテストで固定できる
  • SREのreadinessとbackendの依存関係を接続できる

カバー領域

  • unit test
  • integration test
  • OpenAPI
  • PostgreSQL
  • error contract

Hands-on Flow

  • unit testでservice境界の入力検証を確認する
  • PostgreSQLありのintegration testでSQLiが成立しないことを確認する
  • backend_hands_on_testsでHTTP契約とsecurity regressionを確認する

Docker実行

scripts/learning_phase.sh config p2
scripts/learning_phase.sh start p2
scripts/learning_phase.sh status p2

実行コマンド

npm --prefix app test
npm --prefix app run test:integration
scripts/backend_hands_on_tests.sh
curl -s http://localhost:3000/health/ready | jq .

合格証跡

  • unit/API/integrationのどこで何を守っているか説明している
  • SQLiとpath traversalの再発防止テストが通っている
  • readiness failure時のHTTP契約を記録している
