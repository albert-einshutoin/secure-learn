P7 / Independent 3 / base

Endpoint/EDR観点・OS監査・検知改善

P8に進む前に、ホスト起点の異常を運用改善へ接続できる

WhitehatSRE

抽象的に何を学ぶか

Endpoint/EDR観点・OS監査・検知改善では、個別のコマンド暗記ではなく、プロセス、ファイル、権限、ネットワークをendpoint telemetryとして説明できるための判断軸を学びます。

具体例

  • 具体例: S5、S6を実行し、ファイル改変と権限昇格の監査イベントを確認する
  • 具体例: process treeとevent timelineをincident reportへ入れる
  • 具体例: Sigma/YARA相当の検知条件を疑似ルールとして設計する

到達目標

  • プロセス、ファイル、権限、ネットワークをendpoint telemetryとして説明できる
  • S5/S6のAuditdイベントをSOC caseへ変換できる
  • EDR製品に依存しない検知仮説を作れる

カバー領域

  • auditd
  • process tree
  • privilege escalation
  • case management
  • YARA/Sigma design

Hands-on Flow

  • S5、S6を実行し、ファイル改変と権限昇格の監査イベントを確認する
  • process treeとevent timelineをincident reportへ入れる
  • Sigma/YARA相当の検知条件を疑似ルールとして設計する

合格証跡

  • 誰が、いつ、何を、どの権限で変更したかを説明している
  • endpoint eventをnetwork/API eventと相関している
  • 検知改善案にテストとrollbackがある
