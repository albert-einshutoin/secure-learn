# OSI Learning Map

Docker ラボでは物理層の実配線や無線干渉までは再現しません。代わりに、観測できる層は実測し、観測できない層は「どの証跡がないか」を説明する練習にします。

| OSI層 | 学習レイヤー | 実測シナリオ | 主な観測点 | できるようになること |
|------|--------------|--------------|------------|----------------------|
| L1 | 物理層 | 概念のみ | Docker/ホストのリンク状態 | 物理障害と論理障害を切り分ける質問を作れる |
| L2 | データリンク層 | S8 | ARP, neighbor cache, arping, Suricata flow補助 | MAC/IP 対応と ARP 異常の影響を説明できる |
| L3 | ネットワーク層 | S1, S9 | ICMP, IP, route, Suricata alert | 到達性、経路、スキャン兆候を判断できる |
| L4 | トランスポート層 | S1, S10 | TCP flags, ports, connection state | SYN/FIN/NULL/XMAS とサービス露出を説明できる |
| L5 | セッション層 | S11 | TCP session pressure, incomplete HTTP | 接続維持、枯渇、タイムアウトを運用目線で評価できる |
| L6 | プレゼンテーション層 | S12 | TLS handshake failure, 暗号化境界 | TLS 終端位置と IDS 可視性の制約を説明できる |
| L7 | アプリケーション層 | S2-S4, S13 | HTTP, auth.log, error.log, DNS | 認証、入力、HTTP/DNS を攻撃と検知に結びつけられる |
| OS | ホスト監査 | S5, S6 | Auditd, process, file events | 侵害後操作の証跡を追える |
| 横断 | SOC/SRE | S7, S14, S15 | Kibana, SLO, MTTD/MTTR | 検知から改善までを一連の運用にできる |

## レイヤー別の判断軸

| 判断軸 | 確認すること |
|--------|--------------|
| 攻撃面 | 何が露出しているか、どの入力が悪用されるか |
| 検知面 | どのログに、どのフィールドで、どの粒度で出るか |
| 防御面 | どこで遮断するのが副作用少なく効果的か |
| SRE面 | 可用性、レイテンシ、エラー率にどう影響するか |
| 報告面 | 影響範囲、根拠、暫定対応、恒久対応を説明できるか |
