# Competency Matrix

この表は、受講者が「実行できた」だけで合格しないための評価軸です。各項目は証跡、コマンド、ログ、レポートとセットで確認します。

## ホワイトハット

| レベル | 期待される行動 | 合格証跡 |
|--------|----------------|----------|
| Foundation | 許可された閉域環境だけでスキャン、ブルートフォース、SQLiを再現する | 実行コマンド、対象、時刻、結果 |
| Operator | 攻撃が成功/失敗した理由と、検知されたログを対応付ける | Suricata/Fail2ban/Auditd/Kibana の検索結果 |
| Independent | 検知漏れや誤検知を見つけ、ルールや閾値を安全に調整する | 変更差分、テスト結果、影響範囲 |
| Lead | 脆弱性の修正設計、再発防止、倫理的な開示手順を説明する | 修正案、リスク評価、開示メモ |

## SRE

| レベル | 期待される行動 | 合格証跡 |
|--------|----------------|----------|
| Foundation | `/health`、Compose 状態、基本ログを確認する | `scripts/sre_smoke.sh` の結果 |
| Operator | レイテンシ、エラー率、BAN、アラートを時系列で見る | SLO測定結果、Kibana検索、Fail2ban status |
| Independent | MTTD/MTTR、顧客影響、暫定対応を判断する | インシデントレポート |
| Lead | 再発防止策を設計し、検知・運用・開発の改善をPR化する | 改善PR、テスト、ロールバック方針 |

## 最低合格ライン

- Foundation と Operator は全項目合格
- Independent はホワイトハット、SRE それぞれ2項目以上合格
- S15 の最終レポートで、攻撃、検知、対応、改善を一つのストーリーとして説明できる
