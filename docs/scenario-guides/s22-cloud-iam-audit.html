Cloud IAM/KMS/VPC/Audit Logs

AWS/GCP/Azure共通のIAM、KMS、VPC、Security Group、Audit Logs、Org policyを設計レビューする。

4/5 実務上級 WhitehatSRE

抽象的に何を学ぶか

Cloud securityはサービス名の暗記ではなく、誰が、どのnetworkから、どのkeyで、何を変更し、その証跡が残るかを制御することです。

具体例

  • IAM wildcardを見つけ、action/resource/conditionに分解する。
  • KMS keyのrotation、owner、break-glass、deletion protectionを定義する。
  • CloudTrail/Audit Logsから変更者、source IP、対象resourceをtimeline化する。

目的

cloud accountを実際に触らず、設計レビューとincident evidenceの観点を安全に身につける。

世界レベル評価

実務上級に近い構成です。証跡、修正、運用判断まで扱えます。

Hands-on Flow

  1. Review

    サンプルIAM/Audit logを見て危険な権限と不足ログを指摘する。

  2. Network

    public/private subnet、security group、private endpointを図にする。

  3. Key

    KMS/secret rotationとbreak-glassの手順を書く。

  4. Govern

    Org policy/SCPで禁止すべき操作を定義する。

  5. Report

    cloud incident timelineをincident reportへ転記する。

Whitehat / SRE / Backend 観点

  • Whitehat: 攻撃を許可範囲に限定し、再現性と検知証跡を残す。
  • SRE: SLO、可用性、レイテンシ、復旧判断へ接続する。
  • Backend: サービス露出、ログ、HTTP契約との関係を確認する。
