S9 / L3 / 初級

L3 ICMP到達性・偵察

ping/tracerouteで到達性を確認し、ネットワーク障害とアプリ障害を切り分ける。

3/5 実務中級 WhitehatSRE

抽象的に何を学ぶか

L3 ICMP到達性・偵察は、単発の攻撃手順ではなく、ICMPを偵察と運用監視の両面から評価し、遮断前に必要な監視要件を整理する。

具体例

  • Baseline: 正常時のping、traceroute、/healthを取得する。
  • Execute: /scripts/s9_l3_icmp_recon.shで到達性確認を実行する。
  • Observe: Suricata L3 ICMP alertとKibana searchを確認する。

目的

ICMPを偵察と運用監視の両面から評価し、遮断前に必要な監視要件を整理する。

世界レベル評価

基礎から実務中級の土台を作る構成です。高度領域はS16以降の専門シナリオへ接続します。

Hands-on Flow

  1. Baseline

    正常時のping、traceroute、/healthを取得する。

  2. Execute

    /scripts/s9_l3_icmp_recon.shで到達性確認を実行する。

  3. Observe

    Suricata L3 ICMP alertとKibana searchを確認する。

  4. Triage

    ICMP不通、TCP通、HTTP不調などの切り分け表を作る。

  5. Improve

    監視通信と攻撃偵察を分ける閾値を設計する。

実行コマンド

docker exec -it soc-lab-kali /bin/bash
/scripts/s9_l3_icmp_recon.sh
ping -c 3 app
traceroute app
curl -fsS http://localhost:3000/health

合格証跡

  • ICMP結果とHTTP health結果
  • Suricata alert
  • 不通原因の仮説
  • 監視通信の許可/遮断方針

ツール活用

Tool使い方
ping到達性と往復時間を見る。
traceroute経路とhopの変化を見る。
SuricataICMP/host discoveryの検知を見る。
SRE smokeL3正常でもL7が正常とは限らないことを確認する。

Whitehat / SRE / Backend 観点

  • Whitehat: 攻撃を許可範囲に限定し、再現性と検知証跡を残す。
  • SRE: SLO、可用性、レイテンシ、復旧判断へ接続する。
  • Backend: サービス露出、ログ、HTTP契約との関係を確認する。

世界レベルへ足す課題

  • network SLIを設計し、blackbox monitoringと接続する。
  • ICMP遮断環境の代替probeを追加する。
  • Kubernetes Service/NetworkPolicyの到達性drillを追加する。
