横断インシデント

偵察、認証攻撃、SQLi試行、ファイル/権限イベントを一つのattack chainとして相関する。

4/5 実務上級 WhitehatSREBackend

抽象的に何を学ぶか

横断インシデントは、単発の攻撃手順ではなく、複数レイヤーの検知をtimeline化し、MTTD/MTTR、影響範囲、改善PRまでまとめる。

具体例

  • Start: /scripts/s7_lateral.shで段階的な攻撃チェーンを実行する。
  • Correlate: source.ip、timestamp、event.actionでSuricata/App/Fail2ban/Auditdを繋ぐ。
  • Contain: BANやサービス保護の判断を記録する。

目的

複数レイヤーの検知をtimeline化し、MTTD/MTTR、影響範囲、改善PRまでまとめる。

世界レベル評価

実務上級に近い構成です。証跡、修正、運用判断まで扱えます。

Hands-on Flow

  1. Start

    /scripts/s7_lateral.shで段階的な攻撃チェーンを実行する。

  2. Correlate

    source.ip、timestamp、event.actionでSuricata/App/Fail2ban/Auditdを繋ぐ。

  3. Contain

    BANやサービス保護の判断を記録する。

  4. Eradicate

    成立した攻撃があるか、修正済みで拒否されたかを分ける。

  5. Improve

    検知ルール、backend test、runbookをPRとして提案する。

合格証跡

  • attack chainの開始/検知/対応時刻
  • 各phaseのログソースとKQL
  • 成功した攻撃と拒否された攻撃の区別
  • 改善PRのテストとrollback方針

ツール活用

Tool使い方
Kibana Timeline複数ログソースを時系列でまとめる。

Whitehat / SRE / Backend 観点

  • Whitehat: 攻撃を許可範囲に限定し、再現性と検知証跡を残す。
  • SRE: SLO、可用性、レイテンシ、復旧判断へ接続する。
  • Backend: 入力検証、契約、テスト、エラー安全性を確認する。
