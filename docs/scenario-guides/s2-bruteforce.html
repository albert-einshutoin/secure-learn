APIブルートフォース

Hydraとアプリログを使い、認証攻撃、lockout、Fail2ban、SIEM相関を確認する。

4/5 実務上級 WhitehatBackendSRE

抽象的に何を学ぶか

APIブルートフォースは、単発の攻撃手順ではなく、認証失敗のログ、アプリ側lockout、Fail2ban BAN、正常ユーザー影響を一つの時系列で説明する。

具体例

  • Prepare: docker compose up -d --buildで最新版アプリを起動する。
  • Execute: /scripts/s2_bruteforce.shで低リスクな認証試行を行う。
  • Observe: auth.log、Fail2ban status、Kibana Timelineを確認する。

目的

認証失敗のログ、アプリ側lockout、Fail2ban BAN、正常ユーザー影響を一つの時系列で説明する。

世界レベル評価

実務上級に近い構成です。証跡、修正、運用判断まで扱えます。

Hands-on Flow

  1. Prepare

    docker compose up -d --buildで最新版アプリを起動する。

  2. Execute

    /scripts/s2_bruteforce.shで低リスクな認証試行を行う。

  3. Observe

    auth.log、Fail2ban status、Kibana Timelineを確認する。

  4. Backend check

    AuthServiceのlockout unit testとtoken検証を確認する。

  5. SRE check

    BAN中も/healthと/health/readyがSLO内か確認する。

実行コマンド

docker compose up -d --build
docker exec -it soc-lab-kali /bin/bash
/scripts/s2_bruteforce.sh
docker exec soc-lab-app tail -20 /var/log/app/auth.log
docker exec soc-lab-fail2ban fail2ban-client status nestjs-auth
npm --prefix app test -- --test-name-pattern auth

合格証跡

  • auth.logの失敗回数とsource.ip
  • Fail2banのBAN状態
  • 正常ログインがcredential materialを返さない証跡
  • lockout発動条件と誤検知時のunban手順

ツール活用

Tool使い方
Hydra認証試行を自動化する。対象は必ずラボ内に限定する。
NestJS auth.loglogin_failed/login_successをsource.ipとuser.nameで見る。
Fail2banBAN有無、jail、bantime、findtime、maxretryを確認する。
Node testlockout、token署名、credential material非露出を回帰テスト化する。

Whitehat / SRE / Backend 観点

  • Whitehat: 攻撃を許可範囲に限定し、再現性と検知証跡を残す。
  • SRE: SLO、可用性、レイテンシ、復旧判断へ接続する。
  • Backend: 入力検証、契約、テスト、エラー安全性を確認する。

世界レベルへ足す課題

  • credential stuffing検知として、同一credentialの横展開やlow-and-slow attackを追加する。
  • MFA、risk-based authentication、rate limitのshared store化を設計課題にする。
  • アカウントロックがDoS化しないよう、解除・通知・support flowを追加する。
