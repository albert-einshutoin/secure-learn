eBPF・perf・低レイヤー可観測性

eBPF、perf、flamegraph、syscall telemetryを、性能調査と侵害調査の共通言語として整理する。

3/5 実務中級 SREWhitehat

抽象的に何を学ぶか

eBPF/perfは、アプリログに出ないCPU、syscall、network、file accessを低オーバーヘッドで見るための方法です。抽象的には、ユーザー空間から見えない挙動を安全に観測する技術です。

具体例

  • CPU高騰時、HTTP handlerではなくcrypto、JSON parse、DB client待ちのどこが熱いかを見る。
  • 怪しいprocessがどのfileやnetworkへ触れたかをevent streamとして記録する。
  • 本番ではroot権限とprivacy影響があるため、取得範囲と保存期間を決める。

目的

perf/eBPFを無闇に実行するのではなく、観測したい質問、権限、コスト、プライバシーを定義できるようにする。

世界レベル評価

基礎から実務中級の土台を作る構成です。高度領域はS16以降の専門シナリオへ接続します。

Hands-on Flow

  1. Question

    CPU、syscall、network、file accessのどれを見たいか決める。

  2. Local proxy

    ラボではstrace、ss、lsofで代替観測する。

  3. Design

    本番でeBPFを使う場合の権限、範囲、retentionを設計する。

  4. Correlate

    低レイヤーeventをrequest_idやincident timelineへ繋ぐ。

  5. Control

    観測ツール自体の負荷と機密情報露出を評価する。

実行コマンド

scripts/learning_phase.sh start p10

scripts/world_class_hands_on_check.sh linux

合格証跡

  • 観測したい質問と選んだtoolの理由
  • 低レイヤーeventとアプリ/SLOの対応
  • 権限、負荷、privacyのリスク評価
  • 本番導入時のrollbackと停止条件

ツール活用

Tool使い方
perfCPU sampleとflamegraphでhot pathを見る。
eBPFsyscall、network、file eventを低オーバーヘッドで観測する。
straceラボでsyscall観測を安全に代替する。
Incident timelinekernel/userland eventを時系列に繋ぐ。

Whitehat / SRE / Backend 観点

  • Whitehat: 攻撃を許可範囲に限定し、再現性と検知証跡を残す。
  • SRE: SLO、可用性、レイテンシ、復旧判断へ接続する。
  • Backend: サービス露出、ログ、HTTP契約との関係を確認する。

世界レベルへ足す課題

  • BCC/bpftrace/Cilium Tetragonなどで実イベントを取得する。
  • flamegraphをPRの性能証跡として扱う。
  • EDR telemetryとSRE telemetryの重複/責務を整理する。
