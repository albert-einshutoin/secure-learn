TLS/mTLS・証明書ローテーション

TLS handshake、mTLS、証明書期限、cipher policy、rotationをreleaseとincidentの両面で扱う。

4/5 実務上級 SREBackendWhitehat

抽象的に何を学ぶか

TLS/mTLSは暗号化だけでなく、identity、trust、期限、失効、互換性を運用する仕組みです。抽象的には、通信相手をどう信頼し、その信頼をどう更新するかを扱います。

具体例

  • 証明書期限切れはアプリdeployなしで全リクエストを止める。
  • mTLSではclient証明書の発行、失効、rotationがauthorization境界になる。
  • cipher policy更新は古いclientを切る可能性があるため互換性証跡が必要になる。

目的

TLS/mTLS failureをHTTP障害と分け、rotation、expiry alert、trust boundaryを設計できるようにする。

世界レベル評価

実務上級に近い構成です。証跡、修正、運用判断まで扱えます。

Hands-on Flow

  1. Probe

    openssl s_clientでhandshake情報または失敗理由を見る。

  2. Boundary

    TLS終端前後でIDS/WAF/appが見える情報を分ける。

  3. Design

    mTLSのCA、client cert、rotation、revocationを設計する。

  4. Alert

    expiry、handshake error、protocol downgradeを監視項目にする。

  5. Release

    証明書更新のcanaryとrollbackを決める。

Whitehat / SRE / Backend 観点

  • Whitehat: 攻撃を許可範囲に限定し、再現性と検知証跡を残す。
  • SRE: SLO、可用性、レイテンシ、復旧判断へ接続する。
  • Backend: 入力検証、契約、テスト、エラー安全性を確認する。
