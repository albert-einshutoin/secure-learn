S6 / OS / 中級

権限昇格

sudo/su/SUID探索をAuditdで記録し、最小権限と異常昇格の判断を学ぶ。

3/5 実務中級 WhitehatSRE

抽象的に何を学ぶか

権限昇格は、単発の攻撃手順ではなく、権限昇格を実行できたで終わらせず、許可、証跡、影響、封じ込め、権限設計まで説明する。

具体例

  • Prepare: auditdとprivilege escalation rulesを有効化する。
  • Execute: sudo、su、SUID列挙を安全な範囲で実行する。
  • Observe: audit.logとauth.logでeuid=0、auid、commandを確認する。

目的

権限昇格を実行できたで終わらせず、許可、証跡、影響、封じ込め、権限設計まで説明する。

世界レベル評価

基礎から実務中級の土台を作る構成です。高度領域はS16以降の専門シナリオへ接続します。

Hands-on Flow

  1. Prepare

    auditdとprivilege escalation rulesを有効化する。

  2. Execute

    sudo、su、SUID列挙を安全な範囲で実行する。

  3. Observe

    audit.logとauth.logでeuid=0、auid、commandを確認する。

  4. Decide

    正当運用、要調査、不正昇格に分類する。

  5. Improve

    sudoers最小化、MFA、session recordingを提案する。

実行コマンド

sudo auditctl -l | grep privilege
sudo whoami
find / -perm -4000 2>/dev/null | head
sudo ausearch -k privilege_escalation -i | head -50
sudo aureport --auth --summary

合格証跡

  • 昇格前後のuid/euid
  • auditdのexecve証跡
  • 正当性判断とowner確認
  • sudoers改善案とrollback手順

ツール活用

Tool使い方
sudo/su昇格イベントを意図的に発生させる。
findSUID/SGIDの攻撃面を列挙する。
Auditdeuid、auid、execveを保存する。
sudoers reviewNOPASSWDや広すぎる権限を評価する。

Whitehat / SRE / Backend 観点

  • Whitehat: 攻撃を許可範囲に限定し、再現性と検知証跡を残す。
  • SRE: SLO、可用性、レイテンシ、復旧判断へ接続する。
  • Backend: サービス露出、ログ、HTTP契約との関係を確認する。

世界レベルへ足す課題

  • GTFOBins調査、container breakout防止、Linux capabilityを追加する。
  • PAM/MFA/session recordingの運用設計を追加する。
  • Kubernetes RBAC/PodSecurityと接続する。
