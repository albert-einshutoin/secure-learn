ポートスキャン

nmapで露出ポートを列挙し、Suricata/Kibanaで偵察を検知する基礎シナリオ。

3/5 実務中級 WhitehatSRE

抽象的に何を学ぶか

ポートスキャンは、単発の攻撃手順ではなく、許可されたDockerネットワーク内でactive scanningを実行し、検知、攻撃元特定、誤検知判断、遮断判断までを説明できるようにする。

具体例

  • Prepare: docker compose up -dで環境を起動し、Kaliコンテナへ入る。
  • Execute: /scripts/s1_portscan.sh または nmapのSYN/FIN/XMAS/NULL scanを実行する。
  • Observe: Suricata fast.log/eve.jsonとKibana DiscoverでSCANアラートを確認する。

目的

許可されたDockerネットワーク内でactive scanningを実行し、検知、攻撃元特定、誤検知判断、遮断判断までを説明できるようにする。

世界レベル評価

基礎から実務中級の土台を作る構成です。高度領域はS16以降の専門シナリオへ接続します。

Hands-on Flow

  1. Prepare

    docker compose up -dで環境を起動し、Kaliコンテナへ入る。

  2. Execute

    /scripts/s1_portscan.sh または nmapのSYN/FIN/XMAS/NULL scanを実行する。

  3. Observe

    Suricata fast.log/eve.jsonとKibana DiscoverでSCANアラートを確認する。

  4. Decide

    社内スキャン、監視、攻撃のどれかをsource.ipと時刻で分類する。

  5. Improve

    thresholdや許可済みscannerの扱いを変更し、検知漏れと誤検知の両方を説明する。

実行コマンド

docker compose up -d
docker exec -it soc-lab-kali /bin/bash
/scripts/s1_portscan.sh
docker exec soc-lab-suricata grep SCAN /var/log/suricata/fast.log | tail -20
docker exec soc-lab-suricata jq 'select(.alert)' /var/log/suricata/eve.json | head

合格証跡

  • scan開始時刻、終了時刻、対象IP、実行コマンド
  • Suricata fast.log/eve.jsonの該当行
  • Kibana検索条件とスクリーンショット
  • 遮断する/しない判断と理由

Whitehat / SRE / Backend 観点

  • Whitehat: 攻撃を許可範囲に限定し、再現性と検知証跡を残す。
  • SRE: SLO、可用性、レイテンシ、復旧判断へ接続する。
  • Backend: サービス露出、ログ、HTTP契約との関係を確認する。

世界レベルへ足す課題

  • 承認済みscannerと攻撃者scanを区別するallowlist/threshold設計を追加する。
  • 検知ルール変更をPR化し、before/afterの検知率と誤検知率を比較する。
  • 資産台帳や変更管理と照合する運用判断を課題に加える。
