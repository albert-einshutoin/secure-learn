4/5 実務上級 BackendSRE

抽象的に何を学ぶか

Backend productionは、APIが動くことではなく、変更しても既存clientとデータを壊さないことです。抽象的には、互換性と復旧可能性を設計します。

具体例

  • column renameはadd new column、dual write、backfill、read switch、drop oldの順に分ける。
  • paginationなしの一覧APIは負荷と情報漏洩の両方の問題になる。
  • API versioningではold clientが残る期間とdeprecation policyを決める。

目的

DB/API変更をmigration、contract、transaction、performance、rollbackの観点でreviewできるようにする。

世界レベル評価

実務上級に近い構成です。証跡、修正、運用判断まで扱えます。

Hands-on Flow

  1. Test

    unit/integration/OpenAPI contractを実行する。

  2. Design

    expand/contract migrationとrollback手順を書く。

  3. Concurrency

    race conditionとtransaction境界のred testを設計する。

  4. Scale

    pagination、index、query plan、connection poolの確認項目を作る。

  5. Compat

    API versioningとdeprecation noticeを設計する。
