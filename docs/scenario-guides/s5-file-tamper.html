S5 / OS / 中級

重要ファイル改変

Auditdで重要ファイル変更を記録し、変更者、時刻、対象、復旧判断を追う。

3/5 実務中級 WhitehatSRE

目的

Docker外のホスト監査として、FIM、監査ログ、復旧、postmortemまで扱えるようにする。

世界レベル評価

基礎から実務中級までは有効です。世界レベルには追加課題の実測化が必要です。

Hands-on Flow

  1. Prepare

    Linux hostでauditdを有効化し、audit.rulesを読み込む。

  2. Execute

    安全なtest fileまたは明示許可された重要ファイル操作を行う。

  3. Observe

    ausearch/aureportでPATH/SYSCALL/auidを確認する。

  4. Assess

    変更が正当作業か不正操作かを判断する。

  5. Recover

    差分、バックアップ、immutable/FIM戦略を記録する。

実行コマンド

sudo cp auditd/audit.rules /etc/audit/rules.d/soc-lab.rules
sudo systemctl restart auditd
docker exec -it soc-lab-kali /bin/bash
/scripts/s5_file_tamper.sh
sudo ausearch -k passwd_changes -i | head -50
sudo aureport --file --summary

合格証跡

  • 変更ファイル、auid、実行コマンド
  • 変更前後の差分または復元根拠
  • 正当/不正の判断
  • FIMやbackup/restoreの改善案

ツール活用

Tool使い方
auditdSYSCALL/PATH/auidでOS操作を記録する。
ausearch検知キー、ファイル、時刻で絞り込む。
aureport監査イベントをレポート化する。
Kibana/Filebeatホスト監査ログをSIEMへ流す設計を確認する。

Whitehat / SRE / Backend 観点

  • Whitehat: 攻撃を許可範囲に限定し、再現性と検知証跡を残す。
  • SRE: SLO、可用性、レイテンシ、復旧判断へ接続する。
  • Backend: サービス露出、ログ、HTTP契約との関係を確認する。

世界レベルへ足す課題

  • AIDE/TripwireやeBPF系FIMとの比較を追加する。
  • backup restore drillとRTO/RPO測定を追加する。
  • Linux以外のKubernetes node/host監査へ展開する。
