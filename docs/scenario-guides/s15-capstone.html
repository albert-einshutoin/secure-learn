ホワイトハット/SRE修了課題

S1-S14を組み合わせ、攻撃、検知、修正、運用、報告を一つの成果物にする。

4/5 実務上級 WhitehatSREBackend

抽象的に何を学ぶか

ホワイトハット/SRE修了課題は、単発の攻撃手順ではなく、単発実行ではなく、第三者が再現できるevidence portfolioを作り、PR品質で改善を説明する。

具体例

  • Plan: S8-S13から2つ、S2-S6から2つ、S14を選ぶ。
  • Run: 各シナリオのHTMLに従い、開始/検知/対応時刻を記録する。
  • Correlate: Kibana、logs、scripts reportsを一つのtimelineにまとめる。

目的

単発実行ではなく、第三者が再現できるevidence portfolioを作り、PR品質で改善を説明する。

世界レベル評価

実務上級に近い構成です。証跡、修正、運用判断まで扱えます。

Hands-on Flow

  1. Plan

    S8-S13から2つ、S2-S6から2つ、S14を選ぶ。

  2. Run

    各シナリオのHTMLに従い、開始/検知/対応時刻を記録する。

  3. Correlate

    Kibana、logs、scripts reportsを一つのtimelineにまとめる。

  4. Improve

    1つ以上の検知/運用/backend改善をPR形式で説明する。

  5. Review

    安全性、再現性、検知、対応、改善、報告の全項目でセルフレビューする。

実行コマンド

docker compose up -d --build
scripts/lab_quality_gate.sh

合格証跡

  • 選択シナリオと実行ログ
  • timeline、MTTD、MTTR
  • 検知ルールまたはアプリ/運用改善案
  • セルフレビューと残リスク

ツール活用

Tool使い方
All attack scripts選択したscenarioを安全に実行する。

Templatesincident、remediation、postmortemを成果物化する。
GitHub PR改善をreview可能な単位で提出する。

Whitehat / SRE / Backend 観点

  • Whitehat: 攻撃を許可範囲に限定し、再現性と検知証跡を残す。
  • SRE: SLO、可用性、レイテンシ、復旧判断へ接続する。
  • Backend: 入力検証、契約、テスト、エラー安全性を確認する。

世界レベルへ足す課題

  • CVSS/CWE、MITRE、SLO、rollbackを一つのexecutive summaryへ統合する。
  • peer reviewを前提に、反証可能な証跡だけで主張を組み立てる。
  • 次のissue backlogを優先度付きで作る。
