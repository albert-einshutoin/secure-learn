# S15: 統合キャップストーン

## 概要

| 項目 | 内容 |
|------|------|
| シナリオID | S15 |
| 攻撃名 | End-to-end authorized security and reliability exercise |
| 主レイヤー | 全体 |
| MITRE ATT&CK | Reconnaissance → Initial Access → Impact |
| 検知コンポーネント | Suricata, Fail2ban, Auditd, SIEM, SRE checks |
| 難易度 | 上級 |

---

## 目的

S1-S14 の知識を使い、攻撃、検知、対応、改善、報告を一つの実務成果物としてまとめます。

## 課題

1. S8-S13 から2つ、S2-S6 から2つ、S14 を選んで実行する
2. 各シナリオで開始時刻、検知時刻、対応時刻を記録する
3. 1つ以上の検知ルールまたは閾値を改善し、テストする
4. 1つ以上のアプリ/運用改善案を PR 形式で説明する
5. `docs/templates/incident-report.md` を使って最終レポートを作成する

## 合格条件

| 項目 | 合格基準 |
|------|----------|
| 安全性 | 許可された Docker ラボだけを対象にしている |
| 再現性 | 第三者が同じコマンドで再現できる |
| 検知 | 複数レイヤーのログを相関している |
| 対応 | 暫定対応と恒久対応が分かれている |
| 改善 | 変更案にテストとロールバック方針がある |
| 報告 | 影響範囲、根拠、残リスクが明確 |

## 期待成果物

- 実行ログ
- Kibana 検索条件
- 変更したルール/設定の差分
- SLO 測定結果
- インシデントレポート
- セルフレビュー
